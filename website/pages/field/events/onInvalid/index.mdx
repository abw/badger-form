## `onInvalid`

<Split>
  You can add an `onInvalid` handler to a <FieldLink/> which will be called
  when the field fails validation. The field context will be passed as an
  argument.
  <TryItOut>
    Click one of the fields below and then select the other field without
    entering any text. The <FieldValidateOnBlurLink/> option on the
    <FormLink/> means that the fields will be validated when they lose focus.
    They're both marked as <FieldRequiredLink/> so they will fail validation
    (assuming you haven't entered anything) and the `onInvalid` handler will
    be called.

    Also note the use of <FieldRequiredMessageLink/> to set a custom
    message for the second field.
  </TryItOut>
</Split>

the field context as the first argument with a type of `FieldContextItems`.
