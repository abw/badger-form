## `Input`

The `Input` component is included by the <FieldLayoutLink/> component to
render the input for a field.

You can define your own content for a field to replace the default
<FieldLayoutLink/>, but still use the default `Input` component to render
the correct input for the field type.

## Custom Input

You can define your own component to handle the input for a field
using the `Input` field property.

## The `PictureInput` Component

<Split>
  <Source
    code={PictureInputSrc}
    caption="PictureInput"
    language="jsx"
    expand
  />
  <div>
    Here's the implementation of the `PictureInput` component.

    On line 7 we call `useField()` to get the current field
    `value` and the `onChange` function to set the
    value.  We iterate over a list of animals on line 11 and render the
    `Picture` component for each.  We pass the `selected`
    flag on line 15 to indicate if the animal name matches the current
    `value`.  We also pass a `select`
    function on line 16 which calls the `onChange` handler to select
    the animal.

## The `Picture` Component

<Split>
  <Source
    code={PictureSrc}
    caption="Picture"
    language="jsx"
    expand
  />
  Here's the `Picture` component.  It simply renders a `div` with an `onClick`
  handler to call the `select` function passed as a property.  The `animal`
  CSS class is added, along with `selected` if the animal is currently
  selected.  Inside the `div` we render a picture of the animal in an `img`
  element.
