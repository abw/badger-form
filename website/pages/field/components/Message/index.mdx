## `Message`

The `Message` component is used to add a message underneath a <FieldLink/>.
It's usually added automatically, but it's useful if you're implementing your
own layout for a field.

<Split>
  If you have defined `help` for your field then it will be
  displayed here.  If the field fails to validate then the validation
  error message will be displayed.  In this example we're setting
  an explicit `requiredMessage` to override the default message.
  We're also setting a `validMessage` to provide a custom
  message if the field is valid.
  <TryItOut>
    Try submitting the form without entering a value.  You should see
    the `help` message be replaced by the
    `requiredMessage` error message.
  </TryItOut>
</Split>
